POS API Integration Docs

Open API

Real-time POS Checkpoint

Existing pharmacy POS systems can call IADSS before completing an antibiotic sale. The API returns an Approved or Blocked decision and records the attempt for MOH surveillance.

Endpoint

POST /api/transactions

Example Request

{
  "patientId": "PT-2026-0001",
  "hospitalName": "City General Hospital",
  "prescriberLicense": "VN-MOH-2048",
  "antibiotic": "Amoxicillin",
  "antibioticClass": "Penicillin",
  "dosage": "500mg",
  "quantity": 10,
  "treatmentDurationDays": 5
}

Example Response

{
  "message": "Transaction Approved. Data synced to MOH.",
  "transaction": {
    "status": "Approved",
    "reason": "Prescription verified."
  }
}

Decision Logic

  • Missing required data is blocked.
  • Patient ID, hospital or clinic, and prescriber license must match a valid prescription.
  • Antibiotic class, dosage, quantity, and treatment duration must match prescription limits.
  • Every Approved and Blocked attempt is logged for the MOH Dashboard.
