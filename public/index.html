POS API Integration Docs

Endpoints for external pharmacy POS integrations.

Open API

Real-time POS Checkpoint

External pharmacy POS systems can load a prescription, submit a dispense quantity before checkout, and receive an Approved or Blocked decision.

Authentication

Authorization: Bearer <session-token>

Lookup Endpoint

GET /api/prescriptions/:prescriptionId

Dispense Endpoint

POST /api/transactions

Example Request

POST /api/transactions
Authorization: Bearer <pharmacy-token>

{
  "prescriptionId": "RX-2026-0001",
  "quantity": 5
}

Decision Logic

  • Prescription ID must exist in the national prescription registry.
  • Status must not be Expired, Cancelled, or Fully Dispensed.
  • Dispense quantity must be less than or equal to the remaining quantity.
  • Partial dispensing is allowed and the remaining quantity is updated.
  • Blocked transactions can only be overridden when a pharmacist enters a personal license and reason.
  • Every Approved and Blocked attempt is logged for MOH surveillance.
